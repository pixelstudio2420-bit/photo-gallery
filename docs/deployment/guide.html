เส้นทาง A

🥉 Budget — ~฿300/เดือน

ถูกที่สุด ใช้ provision.sh ติดตั้งเสร็จใน ~15 นาที

Step 1 — เช่า VPS ที่ Contabo
  1. ไปที่ contabo.com → Cloud VPS
  2. เลือก Cloud VPS 10 SSD: 2 vCPU, 8GB RAM, 50GB NVMe (~€6.99/mo ≈ ฿270)
  3. Region: Singapore (latency ต่ำสุดสำหรับผู้ใช้ไทย)
  4. OS: Ubuntu 22.04 LTS
  5. Login Method: Password (เปลี่ยนเป็น SSH key ภายหลัง)
  6. สั่งซื้อ → รอ email ยืนยัน (~15 นาที จะได้ IP + root password)
ทางเลือก: Vultr ($6), DigitalOcean ($6), Hetzner (€4) ก็ใช้ได้
Step 2 — ชี้ DNS ผ่าน Cloudflare
  1. สมัคร cloudflare.com — Free Plan
  2. Add Site → กรอก domain → เลือก Free Plan
  3. Cloudflare จะให้ nameservers 2 ตัว เช่น eva.ns.cloudflare.com
  4. เข้า registrar (Porkbun/Namecheap) → เปลี่ยน nameservers เป็นของ Cloudflare
  5. กลับมาที่ Cloudflare Dashboard → DNS → เพิ่ม records: 
    Type  Name              Content         Proxy
A     yourdomain.com    203.0.113.42    OFF (ตอนแรก)
A     www               203.0.113.42    OFF (ตอนแรก)
  6. ⚠ ปิด Proxy (☁️ สีเทา) ก่อน — เพื่อให้ Let's Encrypt ทำงานได้
Step 3 — SSH เข้าเครื่องแล้วรัน provision.sh

เปิด Terminal (Mac/Linux) หรือ PowerShell (Windows 10+):

# SSH เข้าเครื่อง (ใช้ password จาก email)
ssh root@YOUR_SERVER_IP

# ดาวน์โหลด + รัน provision script
curl -fsSL https://raw.githubusercontent.com/YOUR-REPO/main/docs/deployment/provision.sh -o provision.sh
sudo bash provision.sh

Script จะถามข้อมูลต่อไปนี้:

  • Domain: yourdomain.com
  • Email: admin@yourdomain.com (สำหรับ Let's Encrypt)
  • Git repo URL: https://github.com/YOU/photo-gallery.git
  • Git branch: main
  • Install path: /var/www/photo-gallery
  • MySQL credentials: (auto-suggest password)
  • Provision SSL now?: y

รอประมาณ ~10 นาที — Script จะ:

  1. ติดตั้ง PHP 8.2, MySQL 8, Redis, Nginx, Node 20
  2. Clone repo + composer install + npm build
  3. Run migrations
  4. Install Nginx/Supervisor/cron configs
  5. Run Let's Encrypt
  6. Run smoke test (คาดหวัง 95/95 ✓)
Step 4 — เปิด Cloudflare Proxy

หลังจาก HTTPS ทำงานแล้ว กลับไป Cloudflare Dashboard:

  1. DNS → เปิด Proxy (☁️ สีส้ม) ทั้ง 2 records
  2. SSL/TLS → Full (strict)
  3. SSL/TLS → Edge Certificates → เปิด Always Use HTTPS
  4. Speed → Optimization → เปิด Auto Minify (JS, CSS, HTML)
  5. Caching → Browser Cache TTL → 1 hour
Step 5 — สร้าง Admin user คนแรก 
cd /var/www/photo-gallery
sudo -u www-data php artisan tinker

>>> \App\Models\User::create([
        'name' => 'Admin',
        'email' => 'you@mail.com',
        'password' => \Hash::make('STRONG_PASSWORD'),
        'is_admin' => true,
    ]);
Step 6 — ตั้งค่า External Services ผ่าน Admin Panel

เข้า https://yourdomain.com/admin/settings → กรอก:

  • Google OAuth Client ID + Secret (วิธีสมัคร)
  • LINE Channel Access Token (วิธีสมัคร)
  • SlipOK API Key (วิธีสมัคร)
  • Stripe keys (ถ้าใช้)
✓ จบ Path A — เว็บควรขึ้นแล้วที่ https://yourdomain.com
ถ้าจะ re-deploy ครั้งต่อไป: cd /var/www/photo-gallery && sudo -u www-data ./docs/deployment/deploy.sh
เส้นทาง B

🎛 aaPanel GUI — ~฿400/เดือน

Control Panel ฟรี จัดการทุกอย่างผ่าน browser ไม่ต้องใช้ command line

Step 1 — เช่า VPS (เหมือน Path A)

ใช้ Contabo/Vultr/DO ตามสะดวก — ติดตั้ง Ubuntu 22.04

Step 2 — ติดตั้ง aaPanel (1 คำสั่ง)

SSH เข้าเครื่อง แล้วรัน:

URL=https://www.aapanel.com/script/install_7.0_en.sh && \
wget -O install.sh $URL && \
bash install.sh aapanel

เมื่อติดตั้งเสร็จ จะได้ข้อมูล login:

===================================================
Congratulations! Installed successfully!
===================================================
External URL:  http://203.0.113.42:7800/abc123xyz
Internal URL:  http://10.0.0.5:7800/abc123xyz
Username:      wx4j9k2p

⚠ บันทึก URL + Username + Password ไว้ที่ปลอดภัย (ใช้แค่ครั้งเดียวตอนแรก)
Step 3 — Login aaPanel + ติดตั้ง LNMP Stack
  1. เปิด browser ไปที่ URL ที่ได้
  2. Login ด้วย username/password
  3. เจอหน้าจอ "Recommended installation" ให้เลือก LNMP:
    • Nginx 1.24
    • MySQL 8.0
    • PHP 8.2
    • phpMyAdmin 5.2
    • pure-FTPd (optional)
  4. คลิก One-click install → รอ ~10-15 นาที
Step 4 — ติดตั้ง PHP Extensions ที่ Laravel ต้องการ
  1. เมนูซ้าย → App Store → Installed
  2. หา PHP-8.2 → คลิก Settings
  3. แท็บ Install extensions → คลิก Install ให้ครบ:
    fileinfo redis imagick intl bcmath gd opcache zip mbstring xml
  4. แท็บ Configuration → แก้ค่า: 
    post_max_size = 60M
upload_max_filesize = 50M
memory_limit = 512M
max_execution_time = 300
date.timezone = Asia/Bangkok
  5. Save → Reload PHP-FPM
Step 5 — สร้าง Website
  1. เมนูซ้าย → Website → Add Site
  2. กรอก:
    • Domain: yourdomain.com
    • Aliases: www.yourdomain.com
    • Root: /www/wwwroot/photo-gallery/public ⚠ ต้องเป็น /public
    • PHP Version: PHP-8.2
    • Database: Create → MySQL — ตั้งชื่อ photo_gallery
    • FTP: ไม่ต้องสร้าง (เราจะใช้ Git)
  3. Submit → จดบันทึก MySQL password
Step 6 — Clone repo + ตั้งค่า .env ผ่าน Terminal ในตัว

aaPanel มี Terminal ในตัว (ใช้แทน SSH):

  1. เมนูซ้าย → Terminal (ไอคอนสุดท้าย)
  2. รัน command: 
    # ลบ directory ที่ aaPanel สร้างไว้
cd /www/wwwroot
rm -rf photo-gallery

# Clone repo
git clone https://github.com/YOUR/photo-gallery.git

# ตั้งค่า permissions
chown -R www:www photo-gallery
chmod -R 775 photo-gallery/storage photo-gallery/bootstrap/cache

# ติดตั้ง Composer (ถ้ายังไม่มี)
curl -sS https://getcomposer.org/installer | php
mv composer.phar /usr/local/bin/composer

# Install dependencies
cd photo-gallery
sudo -u www composer install --no-dev --optimize-autoloader

# Install Node 20 (ถ้ายังไม่มี)
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
sudo -u www npm ci && sudo -u www npm run build
Step 7 — ตั้งค่า .env + Migrations ผ่าน File Manager
  1. เมนูซ้าย → Files → navigate ไป /www/wwwroot/photo-gallery
  2. คัดลอก .env.production.example → .env (ขวาคลิก Copy)
  3. Double-click .env → แก้: 
    APP_URL=https://yourdomain.com
DB_DATABASE=photo_gallery
DB_USERNAME=photo_gallery
DB_PASSWORD=MYSQL_PASSWORD_FROM_STEP_5
CACHE_STORE=redis
SESSION_DRIVER=redis
QUEUE_CONNECTION=redis
  4. Save
  5. กลับไป Terminal: 
    cd /www/wwwroot/photo-gallery
sudo -u www php artisan key:generate
sudo -u www php artisan migrate --force
sudo -u www php artisan storage:link
sudo -u www php artisan optimize
Step 8 — ติดตั้ง SSL (คลิกเดียว)
  1. Website → คลิก domain → แท็บ SSL
  2. เลือก Let's Encrypt → Apply
  3. รอ ~30 วินาที → ✓ SSL Active
  4. เปิด Force HTTPS
⚠ สำคัญ: ปิด Cloudflare Proxy ตอนขอ SSL (เปิดหลังจากขอเสร็จ)
Step 9 — ตั้ง Queue Workers ผ่าน GUI
  1. App Store → ติดตั้ง Supervisor Manager
  2. เปิด Supervisor Manager → Add daemon
  3. กรอก: 
    Name:       photo-gallery-worker
User:       www
Command:    php /www/wwwroot/photo-gallery/artisan queue:work --sleep=3 --tries=3 --max-time=3600
Directory:  /www/wwwroot/photo-gallery
Processes:  2
  4. เพิ่มอีก 1 daemon สำหรับ downloads queue: 
    Name:       photo-gallery-downloads
Command:    php /www/wwwroot/photo-gallery/artisan queue:work --queue=downloads --timeout=1800
Processes:  1
Step 10 — ตั้ง Cron สำหรับ Laravel Scheduler
  1. เมนูซ้าย → Cron
  2. Add Task: 
    Name:     Laravel Scheduler
Period:   N minutes — 1 minute
User:     www
Command:  cd /www/wwwroot/photo-gallery && /usr/bin/php artisan schedule:run
Step 11 — ตั้ง Backup อัตโนมัติ
  1. Database → photo_gallery → Backup → Periodic backup → Daily, keep 7
  2. Website → photo-gallery → Backup → Daily, keep 7
  3. (Optional) ผูกเข้ากับ Cloudflare R2 / Google Drive ที่เมนู Cloud Storage
✓ จบ Path B — ทุกอย่างจัดการผ่าน aaPanel GUI ได้ ไม่ต้องเปิด terminal อีก

🚀 Laravel Forge — ~฿700/เดือน

Deploy อัตโนมัติจาก GitHub — Zero-downtime + Rollback ได้

Step 1 — สมัคร Laravel Forge
  1. ไปที่ forge.laravel.com
  2. เลือก Plan: Hobby ($12/mo) — รองรับ 1 server
  3. Signup ด้วย GitHub account
Step 2 — เชื่อม Cloud Provider (DigitalOcean/Vultr/Hetzner)
  1. Forge Dashboard → Server Providers
  2. เลือก DigitalOcean (หรือ Vultr/Hetzner/Linode)
  3. สมัคร DO account → API Tokens → Generate token
  4. Paste token ลง Forge → Save
💡 DigitalOcean Referral: ใช้ m.do.co/c/ ได้เครดิตฟรี $200 ใน 60 วัน
Step 3 — Provision Server ผ่าน Forge
  1. Create Server → เลือก DigitalOcean
  2. Spec:
    • Size: Basic 4GB ($24/mo)
    • Region: Singapore (SGP1)
    • PHP Version: PHP 8.2
    • Database: MySQL 8
    • Post-Provision: (ทิ้งว่างไว้ — Forge ติดตั้งครบแล้ว)
  3. Create Server → รอ ~7 นาที
  4. Forge จะติดตั้ง: Nginx, PHP-FPM, MySQL, Redis, Node, Composer, Supervisor
Step 4 — สร้าง Site + เชื่อม GitHub
  1. Server → Sites → New Site
  2. Domain: yourdomain.com, Project Type: General PHP / Laravel, Web Directory: /public
  3. Create Site → คลิกเข้าไปใน site
  4. Apps → Git Repository
  5. Provider: GitHub, Repository: you/photo-gallery, Branch: main
  6. ✓ Install Composer Dependencies → Install
Step 5 — ตั้งค่า .env
  1. Site → Environment → Edit .env
  2. Forge จะใส่ DB_* ให้อัตโนมัติ — แก้เพิ่ม: 
    APP_URL=https://yourdomain.com
APP_ENV=production
APP_DEBUG=false
CACHE_STORE=redis
SESSION_DRIVER=redis
QUEUE_CONNECTION=redis
FORCE_HTTPS=true
LOG_CHANNEL=daily
  3. Save
Step 6 — Deploy Script + Auto-deploy
  1. Site → App → Deploy Script: 
    cd $FORGE_SITE_PATH
git pull origin $FORGE_SITE_BRANCH

$FORGE_COMPOSER install --no-dev --no-interaction --prefer-dist --optimize-autoloader

$FORGE_PHP artisan migrate --force
$FORGE_PHP artisan config:cache
$FORGE_PHP artisan route:cache
$FORGE_PHP artisan view:cache
$FORGE_PHP artisan event:cache
$FORGE_PHP artisan queue:restart

npm ci --silent
npm run build

( flock -w 10 9 || exit 1
    echo 'Restarting FPM...'; sudo -S service $FORGE_PHP_FPM reload ) 9>/tmp/fpmlock
  2. เปิด Quick Deploy → Install webhook ไป GitHub

1. แยก State ออกจาก App Server

เพราะมี app server หลายตัว — ห้ามเก็บ state บน disk ของ app

  • Sessions: ใช้ Redis (SESSION_DRIVER=redis)
  • Cache: ใช้ Redis (CACHE_STORE=redis)
  • Queue: ใช้ Redis (QUEUE_CONNECTION=redis)
  • File uploads: Cloudflare R2 / S3 (FILESYSTEM_DISK=r2)
  • Logs: forward to Papertrail/Logtail instead of local files
2. Managed MySQL Setup
  1. DO → Databases → Create → MySQL 8 (4GB, Singapore, HA node)
  2. เพิ่ม Trusted Sources → IPs ของ app servers ทั้งหมด
  3. Enable Read Replica (ถ้าต้องการ — ~฿1,500/mo)
  4. ใน Laravel .env: 
    DB_HOST=db-mysql-sgp1-xxx.db.ondigitalocean.com
DB_PORT=25060
DB_DATABASE=photo_gallery
DB_USERNAME=doadmin
DB_PASSWORD=xxx
MYSQL_ATTR_SSL_CA=/etc/ssl/mysql/ca-certificate.crt
  5. ใน config/database.php — เพิ่ม read connection: 
    'mysql' => [
    'read' => ['host' => env('DB_READ_HOST')],
    'write' => ['host' => env('DB_HOST')],
    'sticky' => true,
    ...
]
3. Cloudflare R2 สำหรับ User Uploads
  1. Cloudflare Dashboard → R2 → Create Bucket: photo-gallery-uploads
  2. Manage API Tokens → Create Token → อ่าน+เขียน bucket นี้
  3. Laravel .env: 
    FILESYSTEM_DISK=r2

R2_ACCESS_KEY_ID=xxx

4. Load Balancer + Sticky Sessions
  1. DO → Networking → Load Balancers → Create
  2. Algorithm: Least Connections
  3. Droplets: เลือก app servers ทั้งหมด
  4. Health Check: HTTP/443 path /health (สร้าง route นี้ใน Laravel)
  5. SSL: Upload cert หรือใช้ Passthrough → certbot ที่ instance
  6. Sticky Sessions: Off (เพราะ session อยู่ใน Redis)

เพิ่ม health endpoint ใน routes/web.php:

Route::get('/health', fn() => response()->json([
    'ok' => true,
    'db' => DB::connection()->getPdo() ? 'up' : 'down',
    'redis' => Redis::ping() ? 'up' : 'down',

]));

💾 Backup Strategy (3-2-1 Rule)

3 copies of data, 2 different media, 1 offsite 

# /etc/cron.d/photo-gallery-backup

# 1. Database backup - daily 02:00, keep 30 days
0 2 * * * www-data cd /var/www/photo-gallery && \
    php artisan app:backup --to=r2 --type=database

# 2. Storage backup - weekly Sunday 03:00, keep 12 weeks
0 3 * * 0 www-data cd /var/www/photo-gallery && \
    php artisan app:backup --to=r2 --type=storage

# 3. Full backup - monthly, keep 12 months
0 4 1 * * www-data cd /var/www/photo-gallery && \
    php artisan app:backup --to=r2 --type=full

📈 เมื่อไหร่ควรสเกล?

สัญญาณสาเหตุวิธีแก้
CPU > 70% ต่อเนื่อง PHP-FPM workers ไม่พอ เพิ่ม pm.max_children หรืออัพ CPU
RAM > 85% Cache / queue memory leak เพิ่ม RAM หรือ restart worker บ่อยขึ้น
Response time > 2s DB queries ช้า / N+1 Laravel Telescope → หา slow queries → เพิ่ม index
Queue backlog > 1000 Workers ทำงานไม่ทัน เพิ่ม numprocs ใน supervisor
Downtime > 1 นาที/เดือน Single point of failure ย้ายไป Path D (Load balanced)

❌ 500 Error — "No application encryption key has been specified" 
sudo -u www-data php artisan key:generate --force

❌ 502 Bad Gateway

PHP-FPM ไม่ทำงาน หรือ socket path ผิด

# เช็ค PHP-FPM
sudo systemctl status php8.2-fpm
sudo systemctl restart php8.2-fpm

# เช็ค nginx error log
sudo tail -50 /var/log/nginx/error.log
❌ 413 Request Entity Too Large

อัพโหลดสลิปไม่ได้เพราะไฟล์ใหญ่

# nginx.conf
client_max_body_size 50M;

# php.ini
post_max_size = 60M
upload_max_filesize = 50M

sudo systemctl reload nginx php8.2-fpm
❌ Queue jobs ไม่ทำงาน 
# ตรวจ supervisor status
sudo supervisorctl status

# ถ้า worker ตาย - restart
sudo supervisorctl restart photo-gallery:*

# ดู queue ว่ามี job ค้างเท่าไหร่
php artisan queue:monitor default,downloads --max=1000

# ถ้า job fail เยอะ
php artisan queue:failed
php artisan queue:retry all
❌ SSL renewal ล้มเหลว 
# ทดลอง renew manual
sudo certbot renew --dry-run

# ถ้าติด rate limit - ต้องรอ
# ถ้า HTTP-01 challenge fail - เช็คว่า Cloudflare Proxy อาจ block
# วิธีแก้: ใช้ DNS-01 challenge แทน
sudo certbot certonly --dns-cloudflare \
    --dns-cloudflare-credentials /etc/letsencrypt/cloudflare.ini \
    -d yourdomain.com -d www.yourdomain.com
❌ Google Drive rate limit (quotaExceeded)

Drive API มี limit 1,000 requests/100s/user

  • เพิ่ม caching ที่ event_photos_cache
  • ใช้ exponential backoff retry
  • ขอ quota increase ที่ Google Cloud Console → Quotas

User ยังไม่ Add OA เป็นเพื่อน หรือ LINE user ID ผิด format

โค้ดปัจจุบันจัดการให้แล้ว: log เป็น warning + clear stale line_user_id อัตโนมัติ

❌ Disk เต็มจาก log file

du -sh /var/www/photo-gallery/storage/logs/*

# Force logrotate ทันที
