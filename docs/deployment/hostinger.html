4

เข้า Server (SSH)

⏱ ~2 นาที

มี 2 วิธี — เลือกอันที่สะดวก

🅰 วิธีที่ 1: Browser Terminal (ใน hPanel)

ง่ายสุด ไม่ต้องติดตั้งอะไร:

  1. hPanel → VPS → คลิก server ที่ซื้อ
  2. เมนูซ้าย → Browser terminal
  3. คลิก Connect → browser จะเปิด terminal ให้ใช้
  4. Login ด้วย root + password ที่ตั้งไว้
⚠️ Browser terminal copy/paste ลำบากกว่า SSH จาก desktop — เหมาะกับงานเร็วๆ

🅱 วิธีที่ 2: SSH จาก Desktop (แนะนำ)

🪟 บน Windows

เปิด PowerShell (Win+X → Windows PowerShell) หรือ Windows Terminal:

PS C:\> ssh root@YOUR_VPS_IP
# ครั้งแรกจะถาม "Are you sure you want to continue connecting?"
# พิมพ์ yes + กรอก password ที่ตั้งไว้ใน Step 3.3

🍎 บน Mac / 🐧 บน Linux

เปิด Terminal:

ssh root@YOUR_VPS_IP
✓ ถ้า login สำเร็จจะเห็น prompt: root@srv-xxxxx:~#

Troubleshoot Connection

Errorสาเหตุวิธีแก้
Connection refused VPS ยังไม่ ready รออีก 2-3 นาที ลองใหม่
Permission denied Password ผิด Copy-paste จาก password manager
Host key changed Reinstall OS ทำให้ key เปลี่ยน ssh-keygen -R YOUR_VPS_IP แล้ว ssh ใหม่
Timeout Firewall บล็อก hPanel → Firewall → allow port 22
5

Harden Security (สำคัญ!)

⏱ ~5 นาที
⚠ ทำ step นี้ก่อนเปิดเว็บ! VPS ใหม่จะถูกโจมตีภายใน <1 นาทีหลัง online — bot scan port 22 ตลอดเวลา

5.1 สร้าง non-root user

ห้ามใช้ root ทำงานปกติ — สร้าง user ใหม่:

# สร้าง user ชื่อ deploy (เปลี่ยนได้ตามใจ)
adduser deploy
# จะถาม password — ตั้งใหม่ บันทึกไว้
# ข้อมูลอื่น (full name, phone) กด Enter ผ่าน

# ให้สิทธิ์ sudo
usermod -aG sudo deploy

# ทดสอบ
su - deploy
sudo whoami  # ควรตอบ: root
exit  # กลับเป็น root

5.2 ตั้ง SSH Key (ยกเลิก password login)

บนเครื่องของคุณ (ไม่ใช่ VPS) — สร้าง SSH key ถ้ายังไม่มี:

# เช็คว่ามี key อยู่ไหม
ls ~/.ssh/id_ed25519.pub

# ถ้าไม่มี — สร้างใหม่ (กด Enter ผ่าน 3 คำถาม)
ssh-keygen -t ed25519 -C "your@email.com"

# Copy ไป VPS
ssh-copy-id deploy@YOUR_VPS_IP
# (Windows ที่ไม่มี ssh-copy-id ให้ manual copy)

ทดสอบ:

ssh deploy@YOUR_VPS_IP  # ไม่ควรถาม password

5.3 ปิด password login + root SSH

หลังยืนยัน SSH key ใช้ได้แล้ว (สำคัญมาก!):

# บน VPS ในฐานะ root
nano /etc/ssh/sshd_config

# แก้ 3 บรรทัดนี้ (ค้นด้วย Ctrl+W)
PasswordAuthentication no
PermitRootLogin no
PubkeyAuthentication yes

# Save: Ctrl+O, Enter, Ctrl+X
systemctl restart ssh
⚠ อย่าออกจาก SSH session ตอนนี้! เปิด SSH session ใหม่อีก tab ก่อน (ด้วย deploy user + key) เพื่อยืนยันว่าเข้าได้ ถ้าพลาดจะล็อกตัวเองออก

5.4 เปิด Firewall (ufw)

ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp comment 'SSH'
ufw allow 80/tcp comment 'HTTP'
ufw allow 443/tcp comment 'HTTPS'
ufw enable
ufw status verbose
💡 provision.sh ทำ step 5.4 ให้อัตโนมัติ — ถ้าจะรัน provision.sh ในขั้นตอนถัดไป ข้าม step 5.4 ได้
6

ชี้ DNS มาที่ VPS

⏱ ~3 นาที + รอ propagate

ก่อนรัน provision.sh ต้องให้ domain ชี้มาที่ VPS ก่อน (เพื่อให้ Let's Encrypt ขอ SSL ได้)

6.1 ถ้าใช้ Cloudflare (แนะนำ)

  1. Login dash.cloudflare.com
  2. เพิ่ม domain (ถ้ายังไม่มี) → เลือก Free Plan
  3. เปลี่ยน nameservers ที่ registrar ให้ใช้ของ Cloudflare
  4. DNS → Records → Add: 
    Type: A     Name: @       IPv4: YOUR_VPS_IP   Proxy: OFF (สีเทา)
Type: A     Name: www     IPv4: YOUR_VPS_IP   Proxy: OFF (สีเทา)
⚠ Proxy ต้องปิด (☁️ เทา) ตอนแรก เพื่อให้ Let's Encrypt ยืนยัน domain ownership ได้ — เปิด proxy หลังมี SSL แล้ว

6.2 ถ้าใช้ Hostinger DNS

  1. hPanel → Domains → คลิก domain
  2. DNS / Nameservers → DNS Records
  3. Add record: 
    Type: A     Name: @       Points to: YOUR_VPS_IP     TTL: 3600
Type: A     Name: www     Points to: YOUR_VPS_IP     TTL: 3600

6.3 ตรวจสอบ DNS Propagation

# รอ 2-5 นาที แล้วเช็ค
dig +short yourdomain.com
# ควรคืนค่า: YOUR_VPS_IP

# หรือเช็คออนไลน์
# https://dnschecker.org/#A/yourdomain.com
💡 DNS propagation ปกติ 5-60 นาที — ถ้าเช็คแล้ว IP ตรง ทำ step ต่อไปได้เลย

🎁 Hostinger-Specific Tips

📸 เปิด Auto Snapshots

Hostinger แถม backup รายสัปดาห์ฟรี — ต้องเปิดเอง:

  1. hPanel → VPS → เลือก server
  2. เมนูซ้าย → Snapshots → Create snapshot
  3. ตั้ง schedule: Weekly (ฟรี)
💡 Snapshots ใช้ restore full server ได้ — ใช้แก้ปัญหาถ้า VPS พัง แต่ไม่แทน DB backup ปกติ

🛡 Malware Scanner (Monarx)

ฟรีในทุกแผน KVM — สแกนไฟล์ PHP หา backdoor:

  1. hPanel → VPS → Security → Monarx
  2. Enable → ตั้ง scan daily
  3. แจ้งเตือนผ่าน email ถ้าเจอ malware

📊 Monitoring Graphs

hPanel → VPS → Overview — เห็นกราฟ:

  • CPU usage
  • RAM usage
  • Disk I/O
  • Network in/out

ถ้า CPU > 80% ต่อเนื่อง หรือ RAM เกือบเต็ม → เป็นเวลาอัพเกรดแผน

❌ SSH connect ไม่ได้หลัง reinstall OS

Host key เปลี่ยนเพราะ OS ใหม่:

ssh-keygen -R YOUR_VPS_IP
ssh root@YOUR_VPS_IP

❌ Certbot fails — "Connection timeout"

Cloudflare proxy ยังเปิดอยู่ หรือ DNS ยังไม่ propagate:

  1. ปิด Cloudflare proxy (☁️ เทา)
  2. รอ DNS propagate (5-30 นาที)
  3. เช็ค: dig +short yourdomain.com ตรงกับ VPS IP
  4. รัน certbot ใหม่: sudo certbot --nginx -d yourdomain.com

❌ "Too many requests" จาก Let's Encrypt

โดน rate limit (5 ครั้ง/สัปดาห์ต่อ domain):

  • รอ 7 วัน
  • หรือใช้ staging cert ทดสอบก่อน: certbot --staging

❌ Out of Memory (OOM killer)

KVM 1 อาจมี RAM ไม่พอถ้ามี traffic เยอะ:

# ดู memory usage
free -h

# ดู process ใช้ RAM เยอะ
ps aux --sort=-%mem | head

# เพิ่ม swap (workaround)
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile && swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab

ถ้าเกิดซ้ำ → อัพเกรดเป็น KVM 2

❌ Disk full

# ดูไฟล์ที่กิน disk
du -sh /* 2>/dev/null | sort -h

# ล้าง apt cache
apt clean
apt autoremove

# ล้าง log เก่า
journalctl --vacuum-time=7d
find /var/www/photo-gallery/storage/logs -mtime +30 -delete
