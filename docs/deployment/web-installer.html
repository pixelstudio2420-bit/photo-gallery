ภาพรวม Web Installer

Photo Gallery มีโหมด Install Mode ที่ตรวจจับว่ายังไม่มี admin user แล้วเปิดให้ /admin/deployment เข้าได้ โดยไม่ต้อง login — ผู้ใช้ตั้งค่า server ทั้งหมดผ่าน Web UI

เปรียบเทียบ Web Installer vs CLI

เรื่องWeb Installer (ใหม่)CLI Manual (เดิม)
Skill ที่ต้องมีคลิกเป็นพอSSH + bash
เวลา~15 นาที~30-60 นาที
ข้อผิดพลาดUI guard + test ก่อน savecopy-paste ผิดง่าย
Webhook URLsauto-generate + Copyเขียนเอง
เหมาะกับลูกค้าที่ไม่ technical / re-configdevops / automation
ใช้คู่กันได้: ใช้ provision.sh ติดตั้ง LAMP + nginx + SSL (CLI) แล้วใช้ Web Installer ตั้งค่า DB / Admin / API credentials (Web)

เมื่อไร Install Mode เปิดอัตโนมัติ

ระบบเช็ค 4 เงื่อนไข — เจออันใดอันหนึ่ง = อยู่ใน install mode

Stageเงื่อนไขผล
no_keyAPP_KEY ใน .env ว่างโชว์ปุ่ม Generate APP_KEY
no_dbเชื่อม DB ไม่ได้ชี้ไปแท็บ Database
no_migrationsไม่มี table auth_adminsโชว์ปุ่ม Run Migrations
no_adminไม่มี admin user เลยโชว์ฟอร์มสร้าง admin
installedมี admin ≥ 1 คนauto-secure — ต้อง login

ตรวจสถานะปัจจุบันด้วย CLI (optional)

$ php artisan tinker >>> \App\Services\Deployment\InstallMode::isActive() # true = อยู่ใน install mode >>> \App\Services\Deployment\InstallMode::stage() # "no_admin" / "no_db" / "no_migrations" / "installed"

ข้อกำหนดก่อนเริ่ม

Web Installer ทำได้แค่ส่วน application — ส่วน server-level ยังต้องทำเอง

✅ ทำเองก่อน (server-level)

  • ติดตั้ง PHP 8.2+, MySQL/MariaDB, nginx, Composer, Node.js
  • upload code + รัน composer install --no-dev + npm run build
  • copy .env.example → .env + ตั้ง permissions chmod 664 .env
  • ตั้ง nginx vhost ชี้ไปที่ /path/to/photo-gallery/public
  • ตั้ง SSL (Let's Encrypt) — provider บังคับ HTTPS

👉 ใช้ provision.sh ทำให้อัตโนมัติได้

✅ Web Installer ทำให้ (application-level)

  • Generate APP_KEY
  • ตั้ง DB credentials ใน .env (พร้อม test connection)
  • รัน migrations
  • สร้าง super-admin คนแรก
  • ตั้ง SMTP / R2 / S3 credentials (optional)
  • Show webhook URLs ที่ต้อง register
ห้ามลืม .env writable! Permission ต้อง 664 + owner เป็น www-data ไม่งั้น save ไม่ได้

กดปุ่ม "Generate APP_KEY" — ระบบจะ:

  1. random 32 bytes ผ่าน random_bytes(32)
  2. encode เป็น base64 → base64:xxxxxxxx...
  3. เขียนลง .env แบบ atomic (backup ก่อน)
  4. รัน php artisan config:clear อัตโนมัติ
เทียบเท่ากับ php artisan key:generate แต่ทำผ่านเว็บแบบไม่ต้องใช้ shell

ถ้าใส่ creds ผิดแล้ว save ทันที — เว็บจะใช้งานไม่ได้! ใช้ปุ่ม ทดสอบ เสมอ

Step 3: รัน Migrations

กลับมาที่ Install banner → กด "รัน php artisan migrate" — ใช้เวลาประมาณ 30 วินาที

สิ่งที่เกิดขึ้น

  • เรียก Artisan::call('migrate', ['--force' => true])
  • สร้าง tables ทั้งหมด ~120+ tables
  • seeder default plans + settings ถูก insert อัตโนมัติ
  • output ของ artisan ถูก trim แล้วแสดงในข้อความ flash

ผลลัพธ์ที่ควรเห็น

✓ รัน migrations สำเร็จ — Migrating: 2024_01_15_create_users_table Migrated: 2024_01_15_create_users_table (12.34ms) ...
ถ้าเจอ error SQLSTATE[42S01]: Base table 'X' already exists — แสดงว่า DB เคยรันมาแล้ว ลบ DB แล้วเริ่มใหม่ หรือ skip ขั้นนี้ไปสร้าง admin ต่อเลย

Step 4: สร้าง Super-Admin คนแรก

ฟอร์มสุดท้ายของ wizard — กรอกข้อมูล admin คนแรกของระบบ (super-admin role อัตโนมัติ)

Fieldกฎ
ชื่อ≤ 120 ตัว · split เป็น first_name + last_name
Emailemail format · unique ใน auth_admins
Password≥ 8 ตัว · max 128 ตัว
Confirmต้องตรงกับ password

สิ่งที่เกิดขึ้น

  1. insert row ลง auth_admins ด้วย password hash (bcrypt cost 12)
  2. set role = 'super_admin' + is_active = 1
  3. redirect ไป /admin/login พร้อมข้อความ success
  4. Install Mode auto-end ทันที (มี admin แล้ว)
  5. หน้า /admin/deployment เปลี่ยนเป็น require login
เปิด 2FA ทันทีหลัง login! Settings → 2FA → scan QR ด้วย Google Authenticator / Authy

Tabs อื่น: App / Mail / Storage

หลัง login admin แล้ว กลับมาที่ /admin/deployment เพื่อตั้งค่าเพิ่ม

Tab Application

  • APP_NAME — ชื่อเว็บที่แสดงทุกที่ (browser title, email, etc.)
  • APP_URL — domain ของคุณ (https://...) — สำคัญสำหรับ webhook + OAuth callback
  • APP_ENV — local / staging / production
  • APP_DEBUG — ⚠️ ห้ามเปิดบน production
  • APP_TIMEZONE — Asia/Bangkok

Tab Mail (SMTP)

  • Mailer: smtp / log / sendmail / mailgun / ses
  • SMTP host + port + username + password + encryption (TLS/SSL)
  • From address + From name
  • Test ด้วย Send Test Email ก่อน save

Tab Storage

เลือก 1 ใน 3:

  • Local — storage/app/public (เริ่มต้น)
  • AWS S3 — scalable / pay-as-you-go
  • Cloudflare R2 — ถูกกว่า S3, ฟรี egress (แนะนำ)

R2 preset: region=auto + path-style endpoint=on + endpoint=https://<account-id>.r2.cloudflarestorage.com

Tab Health — System Check

ตรวจสุขภาพระบบทั้งหมด — เปิดดูเป็นประจำเพื่อจับปัญหาล่วงหน้า

4 กลุ่มที่ตรวจ

  • PHP — version + 12 required extensions (pdo, mbstring, gd, ฯลฯ)
  • File Permissions — storage/, bootstrap/cache/, .env writable
  • Database (live) — driver, version, table count
  • Runtime — Cache OK · Storage OK · memory limit · upload max

.env Backups

ทุกครั้งที่ save settings — ระบบ backup .env เก่าไว้ที่ storage/app/env-backups/ เก็บ 30 ไฟล์ล่าสุด (auto-prune)

หลัง Install: Auto-Secure

ระบบ self-locking — ทันทีที่มี admin user 1 คนใน auth_admins:

  • ✅ InstallMode::isActive() return false
  • ✅ middleware admin.or.install เปลี่ยนเป็น require admin login
  • ✅ install endpoints (key/migrate/admin) ปฏิเสธทุก request ที่ไม่ login
  • ✅ /admin/deployment redirect ไป /admin/login ถ้าไม่ login
  • ✅ Install banner หายไป — เห็นเฉพาะ tabs ปกติ
ไม่มี flag ที่ลืมปิด! ระบบใช้การมี/ไม่มี admin row เป็น gate — ลบ admin = install mode กลับมาเปิด

Security Model — Drive-by Protection

Install mode บน production มี IP whitelist กัน drive-by hijack

เปิดได้เฉพาะ:

Pathเงื่อนไขผล
ALocalhost (127.0.0.1, ::1)✅ เข้าได้
BPrivate LAN (10.x, 192.168.x, 172.16-31.x)✅ เข้าได้
CAPP_ENV = local / staging / testing✅ เข้าได้
DLogin admin แล้ว (post-install)✅ เข้าได้
EPublic IP + production + ไม่ login❌ 403 Forbidden + log warning

Threat Model

ถ้า attacker หา URL /admin/deployment เจอตอนคุณกำลัง deploy:

  • local dev — ไม่กระทบเพราะ attacker ไม่อยู่ใน LAN ของคุณ
  • production จาก public IP — โดน 403 ทันที (เพราะ APP_ENV=production)
  • กรณีพิเศษ — admin ใช้ proxy/VPN ที่ไม่ใช่ private IP — ต้อง login ก่อน หรือเปลี่ยน APP_ENV ชั่วคราว
เคล็ดลับ: ทำ Step 1-4 ติดต่อกันให้เสร็จในรอบเดียว (5-10 นาที) เพื่อลด exposure window

Troubleshooting

❌ "Install mode is restricted to local/private network..."

เกิดเมื่อ APP_ENV=production + เข้าจาก public IP. แก้:

  • ssh เข้า server แล้วตั้ง APP_ENV=staging ชั่วคราว
  • หรือ access ผ่าน localhost ของ server (ssh + tunnel)
  • หรือ login admin ก่อน (ถ้ามี admin แล้ว)

❌ ".env file ไม่สามารถเขียนได้"

$ sudo chown www-data:www-data .env $ sudo chmod 664 .env

❌ Migrations ล้มเหลว — "Base table already exists"

# DB เคย migrate แล้ว — drop + recreate $ mysql -u root -p mysql> DROP DATABASE photo_gallery; mysql> CREATE DATABASE photo_gallery CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

❌ "Test connection failed" ที่ Tab Database

  • ตรวจ MySQL service: sudo systemctl status mysql
  • ตรวจ user permissions: SHOW GRANTS FOR 'pg_user'@'localhost';
  • ตรวจ port 3306 เปิด: ss -tnlp | grep 3306
  • ลอง connect ด้วย CLI: mysql -u pg_user -p photo_gallery

❌ admin login ไม่ได้หลังสร้าง

  • ตรวจว่า session storage path writable: chmod 775 storage/framework/sessions
  • ตรวจ APP_KEY ตั้งแล้ว (ดูใน .env)
  • ตรวจ auth_admins table มี row จริง: SELECT email FROM auth_admins;
