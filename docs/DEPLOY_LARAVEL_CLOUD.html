Deployment Guide

Deploy บน Laravel Cloud

คู่มือ step-by-step สำหรับมือใหม่ — ทำตามได้เลย ไม่ต้องมีพื้นฐาน DevOps

~30-60 นาที (ครั้งแรก) · เริ่มได้ฟรี (Hobby plan) · SSL + Auto-scale ไม่ต้องตั้งเอง
Laravel Cloud คืออะไร? บริการ hosting ทางการของ Laravel เอง (เปิดตัวปี 2025) — deploy โดย push git commit เดียวจบ ไม่ต้องตั้งค่า VPS, Nginx, PHP-FPM, supervisor เอง

สิ่งที่ Laravel Cloud จัดการให้อัตโนมัติ

  • Web server (Nginx + PHP-FPM)
  • Queue worker
  • Scheduler (cron)
  • SSL certificate
  • Auto-scaling
  • Zero-downtime deploy
  • Logs streaming

1เตรียมตัวก่อน deploy

สิ่งที่ต้องมี (ติดตั้งใน machine ของคุณ)

git --version       # ต้องมี Git
node --version      # ต้องมี Node.js 18+
php --version       # ต้องมี PHP 8.2+
composer --version  # ต้องมี Composer 2.x

สิ่งที่ต้องสมัคร (ถ้ายังไม่มี)

บริการใช้ทำอะไรเริ่มต้นฟรี
GitHubเก็บ source code✅ ฟรี
Laravel Cloudhosting✅ มี free tier (Hobby)
CloudflareDNS + R2 storage✅ ฟรี (R2 ฟรี 10GB)
LINE DevelopersLINE Messaging API✅ ฟรี
Google CloudOAuth + Drive✅ ฟรี
SlipOKตรวจสลิป PromptPayมีแพ็คเกจฟรีจำกัด
AWSRekognition + S3 (option)มี free tier
Sentryerror tracking (option)✅ ฟรี dev tier
💡 Tip: เริ่มจากของฟรีก่อน เปิดเฉพาะ feature ที่ต้องใช้

2สมัคร Laravel Cloud

  1. ไปที่ https://cloud.laravel.com
  2. กด Sign up → ใช้ GitHub login (แนะนำ)
  3. ยืนยัน email
  4. เข้า dashboard ของ Laravel Cloud

เลือก Plan

Planราคาเหมาะกับ
Hobbyฟรีลองเล่น, MVP, traffic น้อย (sleep หลัง 30 นาที)
Production$20+/moproduction จริง, no sleep, autoscale
Enterpriseสั่งราคาscale ใหญ่
มือใหม่แนะนำ Hobby ก่อน → ถ้า traffic เริ่มเยอะ upgrade เป็น Production

3Push โค้ดขึ้น GitHub

A. สร้าง repo ใหม่

  1. ไปที่ https://github.com/new
  2. Repository name: photo-gallery
  3. Visibility: Private (แนะนำ — มี API keys)
  4. ❌ ไม่ต้องติ๊ก Initialize with README
  5. กด Create repository

B. Push โค้ด

# ถ้ายังไม่ได้ init git
git init
git branch -M main

# เพิ่ม .env เข้า .gitignore (สำคัญ! กันรั่ว secret)
echo ".env" >> .gitignore
echo "node_modules/" >> .gitignore
echo "vendor/" >> .gitignore
echo "storage/logs/*" >> .gitignore
echo "public/build/" >> .gitignore

# Add + commit
git add .
git commit -m "Initial commit"

# ผูกกับ GitHub
git remote add origin https://github.com/YOUR_USERNAME/photo-gallery.git
git push -u origin main

4สร้าง Application

ใน Laravel Cloud dashboard:

  1. กด Create Application
  2. Source: เลือก GitHub → ให้สิทธิ์เข้าถึง repo photo-gallery
  3. Branch: main
  4. Name: photo-gallery-prod
  5. Region: เลือกที่ใกล้ผู้ใช้
    • 🇹🇭 ลูกค้าในไทย → ap-southeast-1 (Singapore)
    • 🇺🇸 ลูกค้า US → us-east-1
  6. กด Create

Laravel Cloud จะ:

  • Clone repo
  • Detect ว่าเป็น Laravel project
  • รัน composer install
  • รัน npm install && npm run build (auto-detect Vite)
⚠️ deploy ครั้งแรกจะล้มเหลว เพราะยังไม่ได้ตั้ง env vars + DB → ปกติ

5ตั้งค่า Database (PostgreSQL)

ระบบนี้ใช้ PostgreSQL 16 (ไม่ใช่ MySQL — มี migration ที่ใช้ Postgres-specific syntax เช่น STRING_AGG, INTERVAL, partial unique indexes)

Option A: Laravel Cloud Postgres (แนะนำ)

  1. ไปที่ tab Resources
  2. กด Add Database
  3. Type: PostgreSQL
  4. Version: 16
  5. Plan: เริ่มที่ Starter (1GB)
  6. Region: เดียวกับ application
  7. กด Create

Laravel Cloud จะตั้ง env vars ให้อัตโนมัติ (DB_CONNECTION, DB_HOST, ฯลฯ)

Option B: External Postgres (Neon.tech / Supabase)

DB_CONNECTION=pgsql
DB_HOST=ep-xxxx.us-east-2.aws.neon.tech
DB_PORT=5432
DB_DATABASE=neondb
DB_USERNAME=postgres
DB_PASSWORD=your-password
DB_SSLMODE=require

6ตั้งค่า Redis (Cache + Queue)

Redis ใช้สำหรับ cache + queue + session — ไม่บังคับ แต่แนะนำมากสำหรับ production

Option A: Laravel Cloud Redis

  1. tab Resources → Add Redis
  2. Plan: Starter
  3. Env vars จะถูก inject อัตโนมัติ

Option B: Upstash Redis (ฟรี 10K commands/วัน)

REDIS_HOST=xxx.upstash.io
REDIS_PORT=6379
REDIS_PASSWORD=xxxx

บังคับใช้ Redis

CACHE_STORE=redis
QUEUE_CONNECTION=redis
SESSION_DRIVER=redis

7ตั้งค่า Storage (R2 หรือ S3)

ระบบเก็บรูปทุกตัวบน object storage — ไม่ใช่ใน server (เพราะ photo file ใหญ่)

แนะนำ: Cloudflare R2 (ถูกกว่า S3 ~10x — egress ฟรี)

  1. สมัคร https://cloudflare.com (ฟรี)
  2. Dashboard → R2 Object Storage → Create bucket
    • Name: photo-gallery-media
    • Location: ASIA
  3. R2 → Manage API Tokens → Create API Token
    • Permission: Object Read & Write
    • Specify bucket: photo-gallery-media
  4. เก็บ Access Key ID, Secret Access Key, Account ID
  5. (option) เปิด Public access ที่ Bucket settings → ได้ public URL: https://pub-xxxxx.r2.dev

ตั้งใน Laravel Cloud env

FILESYSTEM_DISK=r2

AWS_ACCESS_KEY_ID=your-r2-access-key
AWS_SECRET_ACCESS_KEY=your-r2-secret
AWS_DEFAULT_REGION=auto
AWS_BUCKET=photo-gallery-media
AWS_USE_PATH_STYLE_ENDPOINT=false
AWS_ENDPOINT=https://YOUR-ACCOUNT-ID.r2.cloudflarestorage.com
AWS_URL=https://pub-xxxxx.r2.dev
⚠️ R2 ใช้ region = auto (ไม่ใช่ region จริง)

8ตั้งค่า Environment Variables

Application → tab Environment

Env vars ที่บังคับต้องตั้ง

# ── Core ──
APP_NAME="Photo Gallery"
APP_ENV=production
APP_KEY=               # generate ด้านล่าง
APP_DEBUG=false
APP_URL=https://your-app.cloud.laravel.com
APP_LOCALE=th
APP_TIMEZONE=Asia/Bangkok
APP_FALLBACK_LOCALE=en

# ── Logging ──
LOG_CHANNEL=stack
LOG_STACK=single
LOG_LEVEL=warning

# ── Session / Cache / Queue ──
SESSION_DRIVER=redis
SESSION_LIFETIME=120
SESSION_ENCRYPT=true
SESSION_SECURE_COOKIE=true

CACHE_STORE=redis
QUEUE_CONNECTION=redis

# ── Database ──
DB_CONNECTION=pgsql
# DB_HOST, DB_PORT, ฯลฯ จะ auto-inject ถ้าใช้ Cloud Postgres

# ── Storage ──
FILESYSTEM_DISK=r2
AWS_ACCESS_KEY_ID=...
AWS_SECRET_ACCESS_KEY=...
AWS_DEFAULT_REGION=auto
AWS_BUCKET=photo-gallery-media
AWS_ENDPOINT=https://YOUR-ACCOUNT.r2.cloudflarestorage.com
AWS_URL=https://pub-xxxxx.r2.dev
AWS_USE_PATH_STYLE_ENDPOINT=false

Env vars เลือกได้

# ── Mail ──
MAIL_MAILER=ses
MAIL_FROM_ADDRESS=noreply@yourdomain.com
MAIL_FROM_NAME="Photo Gallery"

# ── LINE ──
LINE_CHANNEL_ACCESS_TOKEN=your-token
LINE_CHANNEL_SECRET=your-secret

# ── Sentry ──
SENTRY_LARAVEL_DSN=https://xxx@sentry.io/yyy

SENTRY_TRACES_SAMPLE_RATE=0.1

# ── AI ──

💡 Payment gateway keys (Stripe, Omise, SlipOK, etc.) — ตั้งใน /admin/settings/payment-gateways ใน UI หลัง deploy ไม่ต้องตั้งใน env

Generate APP_KEY

ทำในเครื่องของคุณก่อน push:

cd C:\xampp\htdocs\photo-gallery-pgsql
php artisan key:generate --show
# ได้ output: base64:xxxxx...

Copy ไปใส่ใน Laravel Cloud env → APP_KEY=base64:xxxxx...

⚠️ APP_KEY ใหม่หลัง deploy = ทุก session/encrypted data ถูก decrypt ไม่ได้ — ตั้งครั้งเดียว แล้วอย่าเปลี่ยน

9Deploy ครั้งแรก

  1. tab Deployments → กด Trigger Deployment
  2. Laravel Cloud จะรันอัตโนมัติ:
    • composer install --no-dev --optimize-autoloader
    • npm ci && npm run build
    • php artisan storage:link
    • php artisan migrate --force

Seed ข้อมูลตั้งต้น (สำคัญมาก!)

หลัง deploy สำเร็จ → tab Console (web SSH):

# บัญชี admin default
php artisan db:seed --class=DefaultAccountsSeeder

# settings เริ่มต้น (chat OFF, credits OFF, ฯลฯ)
php artisan db:seed --class=AppSettingsSeeder

# alert rules 16 ตัว
php artisan db:seed --class=DefaultAlertRulesSeeder

# addon catalog
php artisan db:seed --class=AddonItemsSeeder

# pricing plans
php artisan db:seed --class=PricingPackageSeeder

# event categories
php artisan db:seed --class=EventCategorySeeder

# Thai geography (จังหวัด/อำเภอ)
php artisan db:seed --class=ThaiGeographySeeder

# Thai banks
php artisan db:seed --class=ThaiBankSeeder

# (option) ทดสอบ
php artisan db:seed --class=TestPhotographersSeeder
php artisan db:seed --class=TestEventsSeeder

Login ครั้งแรก

ไปที่ https://your-app.cloud.laravel.com/admin/login

Email: admin@photogallery.com
Password: password123
⚠️ เปลี่ยน password ทันที ที่ /admin/admins

10เปิด Queue Worker + Scheduler

Queue Worker

  1. Application → tab Workers
  2. กด Add Worker
  3. ตั้งค่า:
    • Connection: redis
    • Queue: default,notifications,payouts
    • Tries: 3
    • Timeout: 600 (10 นาที)
  4. (option) Workers count: 2-4

Scheduler

  1. tab Scheduler
  2. กด Enable Scheduler
  3. เสร็จ — Laravel Cloud รัน php artisan schedule:run ให้อัตโนมัติทุกนาที

ตรวจการทำงาน

ใน admin → /admin/diagnostics:

  • Queue heartbeat: should be < 5 min ago
  • Scheduler last run: should be < 1 min ago

11ผูก Custom Domain + SSL

Laravel Cloud ให้ subdomain ฟรี (มี SSL อยู่แล้ว) — ถ้าต้องการ domain ตัวเอง:

  1. Application → tab Domains
  2. กด Add Domain → ใส่ photogallery.co.th
  3. Laravel Cloud แสดง DNS records ที่ต้องตั้ง

ตั้ง DNS ที่ Cloudflare

TypeNameContentProxy
CNAME@ (root)your-app.cloud.laravel.comDNS only (gray)
CNAMEwwwyour-app.cloud.laravel.comDNS only
⚠️ ถ้าใช้ Cloudflare proxy (orange cloud) ต้องตั้ง SSL mode เป็น Full (strict)
  1. รอ ~5-15 นาที ให้ DNS propagate
  2. กด Verify → SSL จะ issue อัตโนมัติ (Let's Encrypt)

บังคับ HTTPS

APP_URL=https://photogallery.co.th
SESSION_SECURE_COOKIE=true

12ตั้งค่า Integration ภายนอก

ทุก integration ตั้งใน admin UI (ไม่ใช่ใน env) — ดูคู่มือใน MANUAL_2026.html section 7

ลำดับที่แนะนำ

A. Mail (ลำดับแรกสุด)

แอดมินไม่ได้รับ notification ถ้าไม่ตั้ง mail

MAIL_MAILER=ses
MAIL_FROM_ADDRESS=noreply@photogallery.co.th

ทดสอบใน Console:

php artisan tinker
> Mail::raw('test', fn($m) => $m->to('your@email.com')->subject('Test'));

B. Payment Gateway

ที่ /admin/settings/payment-gateways อย่างน้อยเปิด 1 ตัว:

  • ขั้นต่ำ — แค่ PromptPay: ตั้งบัญชีรับเงิน → ระบบสร้าง QR ให้ → ลูกค้า upload สลิป → admin approve manual
  • แนะนำ — เพิ่ม SlipOK: สมัคร SlipOK → ตรวจสลิปอัตโนมัติ ~3-5 วินาที
  • Production — เพิ่ม Stripe / Omise: support card + Apple Pay + Google Pay

C. LINE Messaging API

  1. สมัคร LINE OA (ฟรี: 200 push/month)
  2. ที่ /admin/settings/line: Channel Access Token + Channel Secret
  3. LINE Developers → Webhook URL: https://photogallery.co.th/api/webhooks/line

D. Google OAuth

  1. console.cloud.google.com → New Project
  2. OAuth Client ID → Web application
  3. Redirect URI: https://photogallery.co.th/auth/google/callback
  4. ที่ /admin/settings/google ใส่ credentials

E. AWS Rekognition (AI Face Search) — ถ้าต้องการ

  1. AWS IAM → Create User → AmazonRekognitionFullAccess
  2. ตั้ง env
  3. /admin/features → เปิด face_search
💡 Skip AI Face Search ก่อนได้ — feature flag ปิด default

15Troubleshooting

Error: "SQLSTATE[08006]: could not connect to server"

สาเหตุ: Database ยังไม่ provision หรือ env vars ไม่ตรง

แก้:

  1. tab Resources → DB status = running?
  2. ตรวจ env: DB_HOST, DB_DATABASE, DB_USERNAME, DB_PASSWORD
  3. ถ้า external DB → ตรวจ firewall whitelist

Error: R2 upload fail

แก้:

php artisan tinker
> Storage::disk('r2')->put('test.txt', 'hello');
> // ถ้า error → ดู error message
  • ตรวจ AWS_ENDPOINT URL — ต้องเป็น https://YOUR-ACCOUNT-ID.r2.cloudflarestorage.com
  • ตรวจ R2 token permission — ต้องมี Object Read & Write

Error: "419 Page Expired" หลัง login

APP_URL=https://photogallery.co.th
SESSION_SECURE_COOKIE=true
SESSION_DOMAIN=.photogallery.co.th

Image upload สำเร็จแต่ดูไม่ได้ (broken image)

  1. R2 Dashboard → bucket → Settings → Public Access → Allow
  2. Copy public URL: https://pub-xxxxx.r2.dev
  3. ตั้ง env: AWS_URL=https://pub-xxxxx.r2.dev
  4. php artisan cache:clear

Queue worker ไม่ process job

  1. Application → Workers → status = Running?
  2. Logs → กรอง [queue] → เห็น Processing job?
  3. ตรวจ env QUEUE_CONNECTION ตรงกับ worker config

Scheduler ไม่รัน

  1. Application → Scheduler → toggle = ON?
  2. Console → php artisan schedule:list → เห็น job ทั้งหมด?
  3. Console → php artisan schedule:run → manual test

LINE webhook ไม่ทำงาน

  1. ตรวจ line_channel_secret ตรงกับ LINE Developers
  2. Webhook URL ต้องเป็น HTTPS
  3. ดู payment_audit_log table → action='line_webhook_failed'

Site ช้า / Timeout

  1. Cache config: php artisan config:cache && php artisan route:cache
  2. Upgrade plan: Hobby → Production
  3. Add Redis ถ้ายังใช้ database driver
  4. CDN — เพิ่ม Cloudflare proxy

เปรียบเทียบ Laravel Cloud vs VPS

สิ่งที่ต้องทำLaravel CloudVPS
Setup server✅ คลิกเดียว❌ install เอง
SSL certificate✅ auto❌ ตั้ง certbot เอง
Auto-scaling✅ มี❌ ไม่มี
Zero-downtime✅ auto❌ ทำเอง
Queue worker✅ UI toggle❌ supervisor
Scheduler✅ UI toggle❌ crontab
Logs✅ stream❌ ssh + tail
ราคา$0-20+/mo$5-10/mo

สิ่งที่ทำต่อหลัง deploy

  1. อ่าน MANUAL_2026.html — คู่มือใช้งานทุก feature
  2. เปลี่ยน admin password — /admin/admins
  3. ตั้ง alert rules — /admin/alerts/rules (มี 16 rules default แล้ว)
  4. เปิด feature ที่ต้องการ — /admin/features
  5. ใส่ branding — site logo, favicon ที่ /admin/settings/general
  6. ตั้ง legal pages — privacy/terms/refund ที่ /admin/legal-pages
  7. เชิญทีมงาน — /admin/admins
  8. Monitor 7 วันแรก — ดู /admin/diagnostics ทุกวัน
