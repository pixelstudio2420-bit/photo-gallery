ภาพรวม — ทำไมต้อง Cloudflare + R2?

โปรเจกต์ Photo Gallery เก็บภาพถ่ายเยอะมาก (ต้นฉบับ + thumbnail + watermark + preview) ถ้าเก็บลงดิสก์เซิร์ฟเวอร์โดยตรงจะเจอ 2 ปัญหา: (1) ค่าดิสก์แพงเมื่อโตขึ้น (2) เซิร์ฟเวอร์รับทราฟฟิกดาวน์โหลดรูปไม่ไหวตอนงานยอดเข้า

🟠

Cloudflare (CDN / DNS / Cache)

รับหน้าเว็บแทน server — แคชหน้า HTML, CSS, JS, รูปภาพที่ edge ทั่วโลก แถม DDoS protection + SSL ฟรี + DNS จัดการง่าย

🟣

R2 Object Storage

เก็บไฟล์รูปต้นฉบับ — API เหมือน AWS S3 เป๊ะ แต่ ไม่มีค่า egress (bandwidth ดาวน์โหลด) ถูกกว่า S3 มากสำหรับไฟล์มีเดีย

💡 สำคัญ: คู่มือนี้เขียนเฉพาะสำหรับโปรเจกต์ photo-gallery-tailwind (Laravel 12) — config ทุกอย่างตรงกับ config/filesystems.php ที่มีอยู่แล้ว (disk ชื่อ r2) ไม่ต้องแก้โค้ด

สิ่งที่ต้องเตรียม

บัญชี Cloudflare Free สมัครด้วยอีเมลธรรมดา ใช้งานฟรีได้ถาวร

Part A

🟠 Cloudflare — CDN, DNS, Cache

1 สร้างบัญชี Cloudflare

  1. เข้า dash.cloudflare.com/sign-up
  2. กรอกอีเมลและรหัสผ่าน (แนะนำใช้อีเมลหลักของธุรกิจ ไม่ใช่ gmail ส่วนตัว)
  3. ยืนยันอีเมลในกล่องข้อความ (2FA แนะนำให้เปิดทันที ที่ My Profile → Authentication)
🔑 Pro tip: เปิด 2FA ด้วย TOTP (Google Authenticator / Authy) ก่อนทำอะไรต่อ เพราะ Cloudflare จะถือ DNS ของคุณ ถ้าโดน takeover = โดนขโมยทราฟฟิกเว็บทั้งหมด

2 Add Site + ย้าย Nameservers

  1. ใน dashboard กด + Add a site
  2. กรอกชื่อโดเมน (เช่น yourgallery.com) แบบไม่มี www และไม่มี https://
  3. เลือกแพ็ก Free → Continue
  4. Cloudflare จะ scan DNS records เดิมของคุณให้อัตโนมัติ — ตรวจสอบให้ครบ (โดยเฉพาะ MX records ถ้าใช้อีเมลโดเมน เช่น Google Workspace)
  5. จดชื่อ nameservers 2 ตัวที่ Cloudflare ให้ (เช่น alan.ns.cloudflare.com / carly.ns.cloudflare.com) — ของทุกคนไม่เหมือนกัน
  6. ไปที่ registrar ที่ซื้อโดเมน → DNS / Nameservers → เปลี่ยนจาก default ของ registrar เป็น Cloudflare
⚠️ รอ propagation: การเปลี่ยน nameservers ใช้เวลา 5 นาที ถึง 24 ชั่วโมง (ปกติในไทย ~15-30 นาที) Cloudflare จะส่งอีเมลยืนยันเมื่อ active

3 DNS Records

ไปที่ Websites → yourgallery.com → DNS → Records ตรวจสอบว่ามี records เหล่านี้ครบแล้ว (ถ้าไม่มีก็กด + Add record):

TypeNameContentProxyหมายเหตุ
A @ IP เซิร์ฟเวอร์
(เช่น 203.0.113.10)		 🟠 Proxied โดเมนหลัก
CNAME www yourgallery.com 🟠 Proxied redirect www → apex
CNAME cdn optional public-<hash>.r2.dev
หรือ public bucket		 🟠 Proxied subdomain สำหรับ R2 (ทำใน Step 8)
TXT @ SPF record (ถ้าส่งอีเมล) ⚪ DNS only อย่า proxy TXT records
🟠 Proxied vs ⚪ DNS only:
Proxied = ทราฟฟิกผ่าน Cloudflare → ได้ CDN / cache / DDoS protection แต่ ซ่อน IP server
DNS only = ทราฟฟิกวิ่งตรงไป server → เปิดเผย IP + ไม่มี cache — ใช้กับ MX, TXT, SRV, และ subdomain ที่ต้องการต่อ SSH

4 SSL / TLS

  1. ไปที่ SSL/TLS → Overview
  2. เลือก Full (strict) — บังคับ encryption ตั้งแต่ browser ถึง origin
    ถ้าเซิร์ฟเวอร์ยังไม่มี SSL cert ของตัวเอง ให้เลือก Full (ไม่ strict) ไปก่อน แล้วค่อยสลับเป็น strict ตอนติดตั้ง Let's Encrypt บน origin เรียบร้อย
  3. ไปที่ Edge Certificates → เปิด:
    • Always Use HTTPS — redirect HTTP → HTTPS อัตโนมัติ
    • Automatic HTTPS Rewrites — แก้ mixed content ให้
    • Minimum TLS Version: 1.2

โปรเจกต์นี้มี middleware EdgeCache ตั้ง Cache-Control header ที่ route /events/{slug} แล้ว (60s browser / 600s edge) — Cloudflare จะเคารพ header นี้อัตโนมัติ ไม่ต้องทำอะไรเพิ่ม

Part B

🟣 R2 Object Storage — เก็บรูปภาพ

6 เปิด R2 & สร้าง Bucket

6.1 เปิดใช้งาน R2

  1. ใน Cloudflare dashboard → sidebar ซ้าย → R2 Object Storage
  2. ครั้งแรกจะขึ้นหน้า Purchase R2 — ต้องใส่บัตรเครดิต/เดบิต
    ✅ ยังฟรีอยู่ถ้าไม่เกิน: 10 GB storage, 1M Class A ops, 10M Class B ops ต่อเดือน
  3. ยืนยันบัตร → ได้ R2 account ID (ต้องใช้ตอนทำ endpoint URL)

6.2 สร้าง Bucket

  1. R2 → Create bucket
  2. ชื่อ bucket — แนะนำ yourgallery-photos
    ⚠️ กฎชื่อ bucket: ตัวเล็ก + ตัวเลข + ขีดกลาง (-) เท่านั้น, 3-63 ตัวอักษร, ห้ามเริ่ม/ลงท้ายด้วยขีด, ชื่อที่ตั้งแล้วเปลี่ยนไม่ได้
  3. Location — เลือก Automatic (Cloudflare เลือกให้) หรือ APAC ถ้าอยากล็อกฝั่งเอเชีย
  4. Default storage class — Standard (ใช้ประจำ) หรือ Infrequent Access (ถูกกว่า แต่ดึงแพง)
    → แนะนำ Standard
  5. กด Create bucket

7 สร้าง API Token

Laravel ต้องใช้ Access Key ID + Secret Access Key เพื่อเขียน/อ่าน bucket ผ่าน S3 API

  1. R2 Overview → แถบ Manage R2 API Tokens (ขวาบน) → Create API token
  2. Token name: photo-gallery-production (ตั้งชื่อให้รู้ว่าใช้ที่ไหน)
  3. Permissions: Object Read & Write (ไม่ต้องให้ Admin Read & Write — อันตราย)
  4. Specify bucket: เลือก Apply to specific buckets only → เลือก bucket ที่สร้างในขั้นที่ 6
    🔒 Security: ห้าม สร้าง token Apply to all buckets ในโปรดักชัน — ถ้า token หลุดจะลบ bucket อื่นได้ทั้งหมด
  5. TTL: ถ้าอยาก rotate ทุกปี ใส่ 1 year / ไม่ใส่ = ไม่หมดอายุ
  6. Client IP Address Filtering (optional): ใส่ IP ของ server เพื่อให้ token ใช้ได้เฉพาะจาก IP นั้น
  7. กด Create API Token
🚨 หน้าจอหลังกด Create: Cloudflare จะโชว์ Secret Access Key ครั้งเดียว — ก๊อปเก็บลงที่ปลอดภัยทันที (password manager) ถ้าปิดหน้าไปแล้วต้องสร้างใหม่
Access Key ID (ก๊อปใส่ .env)

8 Public URL / Custom Domain

ไฟล์รูปต้อง เปิดให้ browser ของผู้ใช้โหลดได้ — มี 2 แบบ เลือกตามความเหมาะสม:

  1. เปิด bucket → Settings tab
  2. ส่วน Public access → R2.dev subdomain → Allow Access
  3. ได้ URL แบบ https://pub-<hash>.r2.dev — นำไปใส่ CLOUDFLARE_R2_URL ใน .env
⚠️ ข้อจำกัด R2.dev:
  • Rate limit — Cloudflare อาจ throttle ถ้ามีทราฟฟิกสูง
  • ไม่มี cache rules / WAF control
  • ห้ามใช้ในโปรดักชันตามข้อกำหนดของ Cloudflare
→ ใช้ได้แค่ตอน dev / staging

แนะนำสำหรับโปรดักชัน — ได้ cache, DDoS, WAF เต็มรูปแบบ ไม่มี rate limit

  1. bucket → Settings → Public access → Connect Domain
  2. ใส่ subdomain ที่ต้องการ เช่น cdn.yourgallery.com หรือ img.yourgallery.com
  3. Cloudflare จะสร้าง CNAME record ให้อัตโนมัติ (ชี้ไปที่ bucket) — กด confirm
  4. รอ ~1-2 นาที ให้ propagate → ทดสอบเข้า https://cdn.yourgallery.com/<test-file>
  5. ใส่ URL นี้ใน CLOUDFLARE_R2_URL ของ .env
✅ ข้อดี custom domain: ไม่มี rate limit, ได้ edge cache (ภาพที่โหลดซ้ำจะเร็วเหมือน static CDN), ใส่ Cache Rules ได้ (เช่น browser TTL 30 วัน), โดเมนอยู่ภายใต้แบรนด์

9 CORS Policy

ถ้ามี JS ฝั่ง browser เรียก R2 ตรง (เช่น direct upload, fetch() ข้ามโดเมน) ต้องตั้ง CORS ให้ bucket

  1. bucket → Settings → CORS policy → Add CORS policy
  2. วาง JSON ด้านล่างนี้ (แทน yourgallery.com ด้วยโดเมนจริง):
[
  {
    "AllowedOrigins": [
      "https://yourgallery.com",
      "https://www.yourgallery.com"
    ],
    "AllowedMethods": ["GET", "HEAD", "PUT", "POST"],
    "AllowedHeaders": ["*"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3600
  }
]
💡 อย่าใช้ "*" ใน AllowedOrigins ในโปรดักชัน — จะเปิดให้เว็บใครก็ได้ embed รูปของคุณได้ (leech bandwidth)
Part C

⚙️ ตั้งค่าใน Laravel

10 แก้ไขไฟล์ .env

เปิดไฟล์ .env ของโปรเจกต์ แล้วแก้ค่าเหล่านี้: 

# ═══ Default disk ═══
FILESYSTEM_DISK=r2

# ═══ Cloudflare R2 credentials ═══
CLOUDFLARE_R2_ACCESS_KEY_ID=xxxxxxxxxxxxxxxxxxxxxxxx
CLOUDFLARE_R2_SECRET_ACCESS_KEY=yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy
CLOUDFLARE_R2_BUCKET=yourgallery-photos
CLOUDFLARE_R2_ENDPOINT=https://<account_id>.r2.cloudflarestorage.com
CLOUDFLARE_R2_URL=https://cdn.yourgallery.com
# (หรือ https://pub-xxx.r2.dev ถ้ายังไม่ได้ตั้ง custom domain)
📌 ทำความเข้าใจแต่ละ key:
CLOUDFLARE_R2_ACCESS_KEY_IDAccess Key ID ที่ได้จาก Step 7
CLOUDFLARE_R2_SECRET_ACCESS_KEYSecret ที่โชว์ครั้งเดียวหลังกด Create token
CLOUDFLARE_R2_BUCKETชื่อ bucket จาก Step 6
CLOUDFLARE_R2_ENDPOINTS3 API endpoint — ใส่ account_id ของคุณแทน <account_id>
CLOUDFLARE_R2_URLPublic URL สำหรับให้ browser ของผู้ใช้โหลดรูป (จาก Step 8)
🔒 ห้าม commit ไฟล์ .env ลง git! (โปรเจกต์นี้มี .env ใน .gitignore แล้ว — แค่เตือนไว้)

หลังแก้ .env เสร็จ ให้ clear cache:

$ php artisan config:clear
$ php artisan cache:clear

11 ทดสอบว่าเชื่อมต่อได้

11.1 Tinker (เร็วที่สุด)

$ php artisan tinker

// ทดสอบเขียน
>>> Storage::disk('r2')->put('test.txt', 'hello r2');
=> true

// ทดสอบอ่าน
>>> Storage::disk('r2')->get('test.txt');
=> "hello r2"

// ทดสอบ URL สาธารณะ
>>> Storage::disk('r2')->url('test.txt');
=> "https://cdn.yourgallery.com/test.txt"

// ลบทิ้ง
>>> Storage::disk('r2')->delete('test.txt');
=> true
✅ ถ้าเห็น true และ "hello r2" แปลว่าเชื่อมต่อได้แล้ว — ใช้งานได้จริง

11.2 ผ่าน Dashboard (เห็นภาพ)

  1. กลับไปที่ Cloudflare R2 → bucket ของคุณ → tab Objects
  2. ควรเห็นไฟล์ test.txt ที่เพิ่งอัปโหลด
  3. ถ้ายังไม่เห็น → กด refresh; ถ้ายังไม่มา → ดู Troubleshooting

11.3 อัปโหลดรูปจริงผ่าน UI

  1. ล็อกอินเป็นช่างภาพ → สร้าง event ใหม่
  2. อัปโหลดรูป 1 ภาพ
  3. เช็คที่ R2 bucket → tab Objects → ควรเห็นไฟล์ในโฟลเดอร์ events/{event_id}/photos/ (ตามโครงสร้างของโปรเจกต์)

12 ย้ายรูปเก่าจาก local → R2 optional

ถ้ามีรูปเก่าใน storage/app/public อยู่แล้ว และต้องการย้ายไป R2 ทั้งหมด:

⚠️ ก่อนย้าย: (1) backup ฐานข้อมูลก่อน (2) path ในตารางอาจต้องอัปเดต ถ้า logic เก่าอ้างถึง disk public โดยตรง (3) ทดสอบใน staging ก่อน

ใช้ AWS CLI (ติดตั้ง S3 adapter กับ R2 endpoint):

# 1. ตั้ง credential profile สำหรับ R2
$ aws configure --profile r2
AWS Access Key ID: <CLOUDFLARE_R2_ACCESS_KEY_ID>
AWS Secret Access Key: <CLOUDFLARE_R2_SECRET_ACCESS_KEY>
Default region: auto
Default output format: json

# 2. sync ทั้งโฟลเดอร์ (dry-run ก่อน!)
$ aws s3 sync ./storage/app/public s3://yourgallery-photos \
    --endpoint-url https://<account_id>.r2.cloudflarestorage.com \
    --profile r2 \
    --dryrun

# 3. เอา --dryrun ออกเมื่อดูแล้วถูก
$ aws s3 sync ./storage/app/public s3://yourgallery-photos \
    --endpoint-url https://<account_id>.r2.cloudflarestorage.com \
    --profile r2

🛠 Troubleshooting

Class "League\Flysystem\AwsS3V3\PortableVisibilityConverter" not found

ขาด package league/flysystem-aws-s3-v3 — ติดตั้งด้วย:

$ composer require league/flysystem-aws-s3-v3:"^3.0"
Error: The bucket does not exist
  • ตรวจชื่อใน CLOUDFLARE_R2_BUCKET ตรงกับ dashboard เป๊ะ (case-sensitive)
  • ตรวจว่า API token ให้ บัง bucket นี้ — บางครั้ง scope ผิด
  • ลอง php artisan config:clear
Error: Access Denied / SignatureDoesNotMatch
  • Secret key อาจถูกก๊อปผิด — สร้าง token ใหม่
  • ถ้า token มี IP filter → ตรวจ IP server ตรงกับ filter ไหม
  • เช็คว่า CLOUDFLARE_R2_ENDPOINT มี https:// นำหน้าและ ไม่มี / ปิดท้าย
รูปอัปโหลดได้ แต่ browser โหลดไม่เห็น (403)
  • Bucket ยังไม่ได้เปิด public access → Step 8
  • CLOUDFLARE_R2_URL ชี้ผิด (ชี้ endpoint แทน public URL)
  • Custom domain ยังไม่ propagate — ลอง dig cdn.yourgallery.com ดูว่า CNAME ผูกถูกไหม
CORS error ใน browser console
ไปที่ bucket → Settings → CORS policy → ใส่โดเมนจริงใน AllowedOrigins (Step 9)
Cloudflare แสดง Error 525 / 526 (SSL handshake failed)
Origin server ยังไม่มี SSL cert → เปลี่ยน SSL mode เป็น Full (ไม่ strict) ชั่วคราว แล้วติดตั้ง Let's Encrypt บน origin ก่อนจะกลับเป็น Full (strict)
เว็บโหลดช้า / Cloudflare ไม่ได้ cache ให้
  • เช็ค response header — ถ้ามี cache-control: no-store Cloudflare จะไม่แคช
  • Dynamic routes (/admin, /api) ไม่ควรถูก cache — ถูกต้องแล้ว
  • ใช้ cf-cache-status header: HIT = ถูก cache, MISS = ไม่ใช่, DYNAMIC = ไม่แคชเลย
