OAuth (Google / Facebook / LINE / Apple)

สถาปัตยกรรมใหม่ (v1.1)

ตั้งแต่เมษายน 2026 credentials ของทุก provider ถูกย้ายออกจาก .env ไปจัดการผ่านหน้า Admin (เก็บใน app_settings table แบบเข้ารหัส) เปลี่ยนค่าได้ทันทีโดยไม่ต้อง redeploy

  • 🔑 หน้าหลัก: /admin/settings/social-auth
  • 📂 Google Drive (ใช้ key ร่วม): /admin/settings/google-drive
  • 💬 LINE Messaging (ใช้ channel ร่วม): /admin/settings/line

Google OAuth + Drive

  1. ไป Google Cloud Console → สร้าง Project
  2. APIs & Services → OAuth Consent Screen → ตั้งชื่อแอป
  3. Credentials → Create OAuth Client ID → Web Application
  4. Authorized redirect URIs (copy จากหน้า admin):
    • http://127.0.0.1:8001/auth/google/callback
    • https://yourdomain.com/auth/google/callback
  5. นำ Client ID + Secret ไปใส่ที่ /admin/settings/social-auth → การ์ด Google
💡 Key นี้ใช้ร่วมกันระหว่าง OAuth Login และ Google Drive Storage — แก้ที่ใดที่หนึ่งพอ

Facebook Login

  1. ไป developers.facebook.com
  2. Create App → Consumer → Add Facebook Login
  3. Settings → Basic: ตั้ง Domain + Privacy Policy URL
  4. Facebook Login → Valid OAuth Redirect URIs (HTTPS เท่านั้น)
  5. นำ App ID + Secret ไปใส่ที่ /admin/settings/social-auth → การ์ด Facebook
🔁 Facebook เป็น provider เดียวที่ยังอ่าน .env แบบ fallback ได้ (FB_APP_ID, FB_APP_SECRET) — สำหรับ fresh install

LINE Login

  1. ไป LINE Developers Console
  2. Create Provider → Channel Type: LINE Login Channel ⚠️ (ไม่ใช่ Messaging API)
  3. LINE Login tab → Callback URL ตรงกับ URI ที่แสดงในหน้า admin แบบ exact match
  4. นำ Channel ID + Secret ไปใส่ที่ /admin/settings/social-auth → การ์ด LINE
💡 Channel ID/Secret ใช้ร่วมกันระหว่าง LINE Login และ LINE Messaging API (แก้ได้ที่ /admin/settings/line)

Apple Sign-In (Optional)

  1. ต้องเป็นสมาชิก Apple Developer Program
  2. Identifiers → สร้าง Services ID (เช่น com.yourdomain.auth)
  3. Configure → Sign in with Apple → เพิ่ม Return URL: https://yourdomain.com/auth/apple/callback
  4. Keys → Create Key → Enable "Sign in with Apple" → ดาวน์โหลด .p8
  5. ใส่ที่ /admin/settings/social-auth → การ์ด Apple (4 ช่อง: Service ID, Team ID, Key ID, Private Key)
⚠️ Apple Sign-In บังคับใช้ HTTPS + domain ต้อง verify แล้ว ใน Apple Developer Console
Local Dev: Google/LINE ใช้ http://127.0.0.1:8001 ได้ตรง ๆ (แต่ LINE ต้องตั้ง APP_URL ให้ตรง) — Facebook/Apple ต้อง HTTPS → ใช้ ngrok หรือ cloudflared tunnel

LINE Messaging API

LINE Notify ปิดตัวเดือน 3/2025 → ระบบเปลี่ยนมาใช้ LINE Messaging API แทน ตั้งค่าที่ /admin/settings/line:

  • Channel Access Token (long-lived / stateless)
  • Channel Secret (webhook signature)
  • Webhook URL (ถ้ารับข้อความจากผู้ใช้)
💡 ถ้าเปิด LINE Login ไว้แล้ว Channel ID/Secret จะ sync อัตโนมัติ — ไม่ต้องกรอกซ้ำ

Photo Delivery Service

ช่องทางรายละเอียดต้องตั้ง
Webแสดงในหน้า /profile/downloadsไม่ต้องตั้ง
Emailส่งอีเมลแนบลิงก์/ZIPSMTP ด้านบน
LINEส่ง Flex Message + CarouselLINE Messaging + ผู้ใช้เชื่อม LINE Login

Admin เลือกได้หลายช่องทางพร้อมกัน — default คือ Web + Email

Nginx + SSL (Let's Encrypt)

Nginx Config

/etc/nginx/sites-available/photo-gallery

server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    root /var/www/photo-gallery/public;

    client_max_body_size 50M;
    index index.php;
    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* { deny all; }

    gzip on;
    gzip_types text/css application/javascript application/json image/svg+xml;
}

เปิดใช้งาน + SSL

sudo ln -s /etc/nginx/sites-available/photo-gallery /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# Let's Encrypt
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
sudo systemctl enable certbot.timer   # auto renewal

Queue Worker + Cron

Supervisor Config

/etc/supervisor/conf.d/photo-gallery-worker.conf

[program:photo-gallery-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/photo-gallery/artisan queue:work database --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
user=www-data
numprocs=2
redirect_stderr=true
stdout_logfile=/var/log/photo-gallery-worker.log
stopwaitsecs=3600
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start photo-gallery-worker:*

Cron (Laravel Scheduler)

sudo crontab -u www-data -e
* * * * * cd /var/www/photo-gallery && php artisan schedule:run >> /dev/null 2>&1

Troubleshooting

500 Server Error หลัง deploy 
tail -f storage/logs/laravel.log
sudo chown -R www-data:www-data storage bootstrap/cache
sudo chmod -R 775 storage bootstrap/cache
CSS/JS ไม่โหลด 
npm run build
php artisan view:clear

ตรวจ APP_URL ตรงกับโดเมนจริง

Table doesn't exist 
php artisan migrate --force
php artisan migrate:status
ส่งเมลไม่ได้

• Gmail ต้องใช้ App Password ไม่ใช่รหัสจริง

• ตรวจ MAIL_ENCRYPTION=tls + port 587

• ทดสอบ: php artisan tinker → Mail::raw('test', fn($m) => $m->to('you@x.com')->subject('t'));

Webhook Stripe ไม่ทำงาน

• ตรวจ STRIPE_WEBHOOK_SECRET ตรงกับ dashboard

• Route /webhook/stripe ต้องไม่ผ่าน CSRF middleware

• ดู storage/logs/laravel.log

Upload รูปแล้วไม่แสดง 
php artisan storage:link
ls -la public/storage       # ต้องเห็น symlink
LINE OAuth: 400 Bad Request "Invalid redirect_uri value"

สาเหตุ: APP_URL ใน .env ไม่ตรงกับ host ที่ browse จริง — LINE เช็ค redirect_uri แบบ exact string match

ต้องตรงกัน 3 จุด:

  • .env → APP_URL
  • URL ใน browser (อย่าผสม localhost กับ 127.0.0.1)
  • LINE Developers Console → Callback URL
php artisan config:clear
php artisan cache:clear
# แล้ว copy URI จาก /admin/settings/social-auth ไปใส่ใน LINE Developers Console

⚠️ ตรวจว่าใช้ LINE Login Channel (ไม่ใช่ Messaging API Channel) — ไม่งั้น 400 ตลอด

ใส่ OAuth ใน .env แล้วไม่ทำงาน

ตั้งแต่ v1.1 credentials ของ Google/LINE/Apple ถูกย้ายจาก .env ไปที่ app_settings table แล้ว

→ ต้องตั้งที่ /admin/settings/social-auth (มีเพียง Facebook ที่ยังอ่าน .env แบบ fallback ได้)

ถ้าเพิ่งอัปเกรดจาก v1.0 ลองย้ายค่าจาก .env ผ่าน tinker:

\App\Models\AppSetting::set('social_auth', 'google_client_id', env('GOOGLE_CLIENT_ID'));
\App\Models\AppSetting::set('social_auth', 'google_client_secret', env('GOOGLE_CLIENT_SECRET'));
\App\Models\AppSetting::set('social_auth', 'line_channel_id', env('LINE_CHANNEL_ID'));
\App\Models\AppSetting::set('social_auth', 'line_channel_secret', env('LINE_CHANNEL_SECRET'));
